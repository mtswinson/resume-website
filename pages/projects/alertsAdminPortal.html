Context

The Alerts Admin Portal was a web-based tool designed to log and manage real-time alerts related to driving conditions—such as weather events, road closures, traffic disruptions, and accidents—within a defined geographic area.

Problem/Goal

General Motors needed a system that allowed advisors to record and monitor events that could affect a customer's driving experience. The tool had to support logging of event type, severity, location, time frame, and geographic scope (including multi-state and multi-county coverage). Advisors needed immediate access to this data to provide informed guidance and improve customer safety.

Process

I approached the project as a full-stack contributor, focusing primarily on frontend development while extending backend services to support new data structures. The stack included React for the UI and API's for data persistence. A key constraint was the need to support complex geographic inputs—such as multiple states and county codes—which required backend schema updates and frontend logic to handle dynamic form behavior.

Feature Breakdown

  • Add Alerts Form
    • This was the core component of the portal, enabling advisors to log new alerts. I built a multi-field form that captured event metadata including type, severity, location, and time range. The form supported advanced geographic input, allowing users to specify multiple states, counties, and county codes. To support this, I modified backend services to accept and process object array-based inputs, updated validation logic, and ensured compatibility with downstream systems.
  • Geographic Scope Handling
    • The original backend was not designed to handle multi-region alerts. I extended the data model and API layer to support nested geographic structures, enabling alerts to span multiple locations. On the frontend, I implemented dynamic field rendering and conditional logic to guide the advisor through valid input combinations, reducing the risk of malformed data.
  • Alerts Table Interaction
    • I developed a flexible table interface for viewing and managing logged alerts. Advisors could open a popover to customize the table layout—dragging and dropping column headers, locking or hiding fields, and preserving two fixed-position columns for context. The table state was reactive, automatically updating to reflect layout changes and maintaining data integrity across views.

Impact/Differentiator

The portal gave advisors a centralized way to log and access real-time driving alerts, improving their ability to assist customers proactively. The geographic input enhancements and table customization features made the tool more adaptable to complex scenarios. The drag-and-drop column logic I implemented improved usability and was later referenced in other internal tools.

Reflection/Key Takeaway

This project gave me a strong foundation in full-stack development. While the bulk of my work was frontend-focused, I gained experience in extending backend services to support new data formats and workflows. It prepared me for future projects that required deeper integration between UI behavior and backend logic.
